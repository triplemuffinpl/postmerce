# Roadmap

## Stage 1 - Foundation

- Repository structure.
- Strict TypeScript.
- App and marketing shells.
- PostgreSQL migrations.
- Healthcheck.
- Worker and dry-run publisher contracts.
- Agent documentation.

## Stage 2 - Media

- Video upload.
- Local storage.
- FFprobe metadata.
- Thumbnail generation.
- Media list and details.

## Stage 3 - Posts And Targets

- Create draft posts.
- Add per-platform targets.
- Platform options.
- Schedule now/later.
- Post detail screen.

## Stage 4 - Queue

- Enqueue publish jobs.
- Worker locking.
- Retry policy.
- Manual retry/cancel.
- Worker heartbeat.

## Stage 5 - First Real Adapter

Prefer YouTube or LinkedIn after API feasibility is checked against official docs.

- OAuth.
- Encrypted token storage.
- Refresh.
- Publish.
- Status and logs.

## Stage 6 - More Platforms

- Second adapter to prove the platform abstraction.
- Meta: Instagram Reels and Facebook Page.
- TikTok after API review constraints are clear.

## Stage 7 - SaaS Readiness

Only after the private product is useful:

- users,
- workspaces,
- Stripe/Paddle billing,
- onboarding,
- customer support flows,
- public product analytics.
