# Roadmap

## Stage 1 - Foundation

- Repository structure.
- Strict TypeScript.
- App and marketing shells.
- PostgreSQL migrations.
- Healthcheck.
- Worker and dry-run publisher contracts.
- Agent documentation.

## Stage 2 - Media

- Video upload. Done in foundation stage 2.
- Local storage. Done in foundation stage 2.
- FFprobe metadata. Done in foundation stage 2.
- Thumbnail generation. Done in foundation stage 2.
- Media list and details. Done in foundation stage 2.

## Stage 3 - Posts And Targets

- Create draft posts. Done in stage 3.
- Add per-platform targets. Done in stage 3.
- Platform options. Done in stage 3.
- Schedule metadata. Done in stage 3.
- Post detail screen. Done in stage 3.

## Stage 4 - Queue

- Enqueue publish jobs.
- Worker locking.
- Retry policy.
- Manual retry/cancel.
- Worker heartbeat.

## Stage 5 - First Real Adapter

Prefer YouTube or LinkedIn after API feasibility is checked against official docs.

- OAuth.
- Encrypted token storage.
- Refresh.
- Publish.
- Status and logs.

## Stage 6 - More Platforms

- Second adapter to prove the platform abstraction.
- Meta: Instagram Reels and Facebook Page.
- TikTok after API review constraints are clear.

## Stage 7 - SaaS Readiness

Only after the private product is useful:

- users,
- workspaces,
- Stripe/Paddle billing,
- onboarding,
- customer support flows,
- public product analytics.
